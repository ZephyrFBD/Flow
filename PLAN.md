## 流式架构 (Streaming)

所有 LLM 交互均通过 **SSE (Server-Sent Events)** 流式传输，前端逐步渲染。

### 树流式生成流程

```
用户提交 Prompt/PDF
       ↓
POST /api/v1/knowledge-tree/generate  (Accept: text/event-stream)
       ↓
后端调 LLM 流式 API (openai/claude/ollama 均支持 streaming)
       ↓
每隔 N 个 token / 每完成一个节点，推送 SSE 事件:
  event: node_added
  data: {"node": {...}, "path": ["id1", "id2"]}

  event: node_completed
  data: {"node": {...}}

  event: done
  data: {"tree_id": "..."}

  event: error
  data: {"message": "..."}
       ↓
前端逐步收到事件 → 树组件渐进式渲染
第1秒:     根节点出现
第2-3秒:   子节点逐个出现
第4-5秒:   叶子节点填充
用户可:    在树未完全生成时就点击已出现的节点查看内容
```

### 答题流式反馈流程

```
用户提交答案
       ↓
POST /api/v1/questions/{id}/submit  (Accept: text/event-stream)
       ↓
LLM 流式输出评判结果:
  event: judgment_start
  data: {"type": "choice", "correct": true}

  event: explanation_chunk
  data: {"text": "矩阵乘法不满足交换律，因为..."}

  event: explanation_chunk
  data: {"text": "接着补充说明..."}

  event: done
       ↓
前端逐字显示解析内容，打字机效果
```

### LLM 接口变更

```python
class BaseLLM:
    # 原有: 一次性返回完整 JSON
    async def generate(self, prompt: str, system_prompt: str) -> str: ...

    # 新增: 流式生成，yield 原始 token 块
    async def generate_stream(self, prompt: str, system_prompt: str) -> AsyncIterator[str]: ...
```

### 前端 SSE 消费

```typescript
// 使用 fetch + ReadableStream 消费 SSE
const response = await fetch('/api/v1/knowledge-tree/generate', {
  method: 'POST',
  body: formData,
  headers: { 'Accept': 'text/event-stream' }
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  // 解析 SSE 事件 → 更新树状态
  parseSSEEvents(text, (event) => {
    if (event.type === 'node_added') {
      addNodeToTree(event.data.node, event.data.path);
    }
  });
}
```

### 三种状态的渲染

| 状态 | 树生成页 | 答题页 |
|------|---------|-------|
| **加载中** | 树节点逐个出现 (渐进式) | 评测文字逐字显示 (打字机) |
| **完成** | 完整树 + 完成提示 | 完整反馈 + 下一题按钮 |
| **错误** | 已生成的树保留 + 错误提示 (可重试剩余部分) | 已显示的反馈保留 + 错误提示 |

## 核心数据模型

```python
# 单个知识点节点
class KnowledgeNode:
    id: str
    title: str
    description: str
    keywords: list[str]
    completed: bool = False        # ☑ 学习完成复选框
    children: list[KnowledgeNode]

# 一次 Prompt 记录
class PromptRecord:
    timestamp: str
    prompt_text: str | None        # 用户输入的 Prompt
    pdf_filename: str | None       # 上传的 PDF 文件名
    mode: str                      # "generate" | "refine"

# 出题配置
class QuestionConfig:
    types: list[str]               # ["choice", "essay"] 选择题和/或解答题
    count: int = 5                 # 生成数量
    difficulty: str = "medium"     # "easy" | "medium" | "hard"
    extra_requirements: str | None # 额外要求 (自由文本)

# 例题
class ExampleQuestion:
    id: str
    node_id: str                   # 关联的知识点节点
    type: str                      # "choice" 或 "essay"
    question: str
    options: list[str] | None      # 选择题选项
    answer: str                    # 参考答案
    explanation: str               # 解析
    difficulty: str                # "easy" | "medium" | "hard"
    user_answer: str | None        # 用户的回答
    is_correct: bool | None        # 是否正确
    ai_score: float | None         # 解答题 AI 评分 (0-10)

# ★ 核心: 单文件数据结构 (.knowtree)
class KnowledgeTreeFile:
    format_version: str = "1.0"
    tree_id: str
    title: str
    
    # 树衍生 (版本链)
    version: int = 1
    parent_id: str | None          # 从哪棵树衍生而来
    children_ids: list[str]        # 由此衍生的子树
    
    # 完整数据
    prompt_history: list[PromptRecord]  # 所有 Prompt 记录
    nodes: KnowledgeNode                # 带完成状态的树
    questions: list[ExampleQuestion]    # 所有例题 + 用户作答
    
    created_at: str
    updated_at: str
```

## 关键设计

### 1. 输入组合
Prompt 文本 + PDF 文件 **可任意组合** 提交：
- 仅 Prompt → 从文本生成
- 仅 PDF → 从 PDF 内容生成
- Prompt + PDF → PDF 为主体内容，Prompt 为侧重点提示

### 2. 树精炼 (增量修改)
已有树 + (新 Prompt/PDF) → LLM 理解原结构 → 输出更新后的树
- 精炼不限次数，每次基于上次结果
- 前端提供"用 AI 修改"入口

### 3. ☑ 节点完成复选框
- 每个节点自带 `completed: bool`
- 前端 Tree 节点前显示 Checkbox
- 点击勾选/取消，实时更新
- 根节点可显示整体完成进度 (已完成/总节点数)

### 4. 🌳 树衍生
- 精炼时可以选择**原地更新**或**衍生为新版本**
- 衍生 = 原树作为 parent，创建新的独立分支
- 原树不受影响，形成版本链
- UI 可切换查看不同版本

### 5. 例题数量可选
- 生成时前端有数字输入框 (默认5题)
- API 接受 `count` 参数

### 6. ★ 单文件持久化 (.knowtree)
- **所有数据存储在一个 `.knowtree` JSON 文件中**
- 包含: prompt_history + 树(含完成状态) + 例题(含用户作答)
- 文件可随意移动、备份、分享
- 打开 = 从文件加载全部数据
- 保存 = 将完整状态写入文件 (原地覆盖或另存为)
- 无需 SQLite 数据库 (简化部署)

### 关键 API 详解

**POST /api/v1/knowledge-tree/generate** (SSE 流式)
```
Request: multipart/form-data
  - prompt: string (可选)
  - pdf_file: file (可选)
  - llm_provider: string (可选)

Response: text/event-stream
  event: node_added     # 新节点出现
  event: description    # 节点描述填充
  event: progress       # 生成进度: {"percent": 60}
  event: done           # 生成完成: {"tree_id": "xxx"}
  event: error          # 错误: {"message": "..."}
```

**PUT /api/v1/knowledge-tree/{id}/refine** (SSE 流式)
```
Request: multipart/form-data
  - prompt: string (可选)
  - pdf_file: file (可选)
  - mode: "inplace" | "derive"

Response: text/event-stream
  事件同 generate，但基于已有树:
  event: node_added / node_updated / node_removed
```

**POST /api/v1/questions/generate** (SSE 流式逐题输出)
```
Request: JSON
  - tree_id: str
  - node_id: str (可选)
  - types: ["choice", "essay"]
  - count: int (默认5)
  - difficulty: "easy"|"medium"|"hard"
  - extra_requirements: str (可选)

Response: text/event-stream
  event: question_added   # 一道题目生成完毕，立即显示
  event: done             # 全部完成
```

**POST /api/v1/questions/{id}/submit** (SSE 流式评判反馈)
```
Request: JSON
  - user_answer: string

Response: text/event-stream
  event: judgment_start    # {"type":"choice","correct":true} / {"type":"essay"}
  event: explanation_chunk # {"text":"..."} 逐块输出，前端打字机效果
  event: essay_feedback    # (仅解答题) {"score":8,"suggestion":"..."}
  event: done
```

**POST /api/v1/project/save**
```
Request: JSON
  - tree_id: str
  - file_path: str (可选，默认下载)

Response: .knowtree 文件下载
```

**POST /api/v1/project/load**
```
Request: upload .knowtree file

Response: KnowledgeTreeFile (完整恢复)
```

## 多页面系统设计

共 **7 个核心页面**，使用 React Router 管理路由。

核心交互流程：
```
首页 → 生成页 → 知识树页 → 知识点详解页 → 答题页
                          ↘ 管理页
```

### 页面总览

```
/                       → 首页 (Dashboard)
/generate               → 生成页 (Prompt+PDF输入→生成树)
/tree/:treeId           → 知识树页 (全页树视图 + ☑复选框)
/node/:treeId/:nodeId   → 知识点详解页 (讲解 + 出题选项)
/practice/:questionId   → 答题页 (选择题/解答题作答)
/manage/:treeId         → 管理页 (版本衍生 + 节点编辑)
/settings               → 设置页 (LLM配置 + 主题)
```

### 页面间导航流程

```
首页 (/) ───→ 生成页 (/generate) ───→ 知识树页 (/tree/:id)
                   ↑                        │
                   │                   ┌────┴────┐
                   │                   │         │
                   │             知识树页    管理页 (/manage/:id)
                   │                   │
                   │                   ▼
                   │           知识点详解页 (/node/:treeId/:nodeId)
                   │                   │
                   │                   ▼
                   └─── 答题页 (/practice/:questionId)
```

**关键路径**: 首页 → 生成页 → 知识树页 → 知识点详解页 → 答题页

**精炼/衍生路径**: 知识树页 → [用AI修改] → 生成页(带上下文) → 返回知识树页
**管理路径**: 知识树页 → [管理] → 管理页 → [返回] → 知识树页
│                                                         │
│  ┌─ 生成结果预览 ────────────────────────────────────┐  │
│  │  (生成中...) 进度条 / Skeleton 加载动画            │  │
│  │  (生成完成) 树结构预览 + [📖 进入学习] 按钮        │  │
│  │  (生成失败) 错误提示 + [重试] 按钮                 │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

- LLM 模型选择器 (切换 OpenAI / Claude / Ollama)
- Prompt 文本框（可选）
- PDF 文件拖拽上传（可选）
- 生成时附带讲解复选框
- 生成中 → 进度/骨架屏
- 生成完成 → 树结构预览 + "进入学习" 按钮
- 可从此页跳转到学习页

## 验证方案

1. **Step 1**: 后端健康检查通过，前端页面正常加载
2. **Step 2 (核心闭环)**:
   - ★ 输入 Prompt → 树节点**逐个流式出现**，不等全部生成即可点击已出现的节点
   - 流式中断测试: 生成到一半断网 → 已生成的节点保留 → 可重试继续
   - 上传 PDF 生成树 (同样流式)
   - Prompt + PDF 同时提交
   - 精炼: 原地更新 vs 衍生为新版本，同样流式输出
   - 勾选/取消节点完成状态
   - 切换查看树的不同衍生版本
3. **Step 3**: 保存为 .knowtree → 关闭页面 → 加载 → 数据完整恢复
4. **Step 4**: 上传 50+ 页 PDF，验证分块和 Map-Reduce
5. **Step 5**: Mermaid 示意图渲染
6. **Step 6**: 
   - 指定数量/难度/题型生成例题 → 题目逐个流式出现
   - 答题 → 提交后答案评判**逐字流式显示** (打字机效果)
   - 解答题 AI 评分 + 改进建议流式输出
   - 错题复习
